## 2. Architecture & Code Structure
The EGY project is a sprawling, purely frontend-heavy macro-architecture distributed across multiple decoupled modules:

### Frontend Architecture
- **Next.js 16 App Router**: Powers the main `egyptian-magic-build` and `egyptian-library`, structurally separating global layouts from highly interactive client-side pages.
- **Data-Driven Injection**: The `egyptian-standalone-showcase` utilizes a pure HTML/Vanilla JS pattern where dense strategic rules (`data.js`) are dynamically injected into the DOM by `app.js`.
- **Componentized Micro-Apps**: Features like the DeepReal UI are built as isolated React components (`deepreal-tripillar-ui.tsx`), making them highly portable across the ecosystem.

### Backend & Build Architecture
- **Minimalist SSG**: The `landing-page-showcase` utilizes a custom, dependency-free Node.js static site generator (`build.js`) that injects markdown into templates.
- **API Swarms & AI Backend**: While mostly frontend, the system connects to external LLMs (via Vercel AI SDK) and fact-checking APIs (OpenAlex, EuropePMC) mapped in `api-swarm.ts` to perform deep-real verification.

## 3. Tools, Frameworks, & Deployment Mechanics
- **Core Frameworks**: React 19, Next.js 16, HTML5, Vanilla JavaScript.
- **Styling**: Tailwind CSS v4, custom CSS variables (Dark Luxury themes), glassmorphism (backdrop filters).
- **3D & Animation**: `@react-three/fiber`, `@react-three/drei`, Three.js (r128), Framer Motion, GSAP (ScrollTrigger).
- **Icons & Processing**: `lucide-react`, `marked.js`.
- **Deployment Mechanics**: Configured for edge environments like Vercel, leaning heavily on Static Site Generation (SSG) and client-side rendering for maximum performance and zero cold starts.
